 1. (최초 1회) 깃 초기화 & 첫 커밋
# 1. 현재 폴더에서 깃 저장소 만들기 (최초 1회)
git init

# 2. 변경된 모든 파일 스테이지에 올리기
git add .

# 3. 커밋(설명 메시지 입력)
git commit -m "첫 커밋: 프로젝트 초기화"

# 4. (원격 저장소 연결시) 깃허브 주소 등록
git remote add origin https://github.com/내아이디/내저장소.git

# 5. (최초 푸시, 브랜치 지정 필수)
git push -u origin main   # (또는 master)

✅ 평소 자주 쓰는 커밋 명령어
# 1. 변경 파일(새로 만든 파일/수정한 파일) 스테이징
git add .

# 2. 커밋(메시지는 본문 요약)
git commit -m "버그 수정: 진입조건 로직 개선"

# 3. 원격 저장소로 푸시
git push

💡 중간에 자주 사용하는 명령어
# 변경내역 보기(커밋 전)
git status

# 파일별 변경점 확인
git diff

# 커밋 히스토리 보기
git log --oneline
